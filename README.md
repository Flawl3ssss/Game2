# Снежный Разгон

Мобильная 3D веб-игра с бесконечным процедурным склоном, созданная для дальнейшей публикации в Яндекс Играх.

## Возможности текущей версии

- управление пальцем, мышью и клавиатурой;
- механика зарядки стартового импульса;
- бесконечные детерминированные чанки;
- повторное использование геометрии и `InstancedMesh` для объектов;
- плавающая точка начала координат;
- трамплины, ускорители, монеты и препятствия;
- улучшения разгона, санок и дохода;
- локальные и облачные сохранения;
- интеграция `LoadingAPI` и `GameplayAPI`;
- rewarded-реклама с локальным тестовым режимом;
- автоматическое снижение качества при падении FPS;
- русский и английский интерфейс с выбором языка через SDK.

## Запуск

```bash
npm install
npm run dev
```

Откройте адрес, который покажет Vite. Для проверки на телефоне используйте адрес компьютера в локальной сети.

## Сборка

```bash
npm run build
```

Готовые файлы появятся в `dist/`. Для загрузки в Яндекс Игры упакуйте **содержимое** папки `dist`, чтобы `index.html` находился в корне ZIP.

## Управление

- телефон: проведите пальцем влево или вправо;
- компьютер: `A/D` или стрелки;
- старт: удерживайте экран или `Space`, затем отпустите;
- пауза: кнопка в правом верхнем углу или `Escape`;
- отладочная панель: клавиша `` ` ``.

## Архитектура бесконечной карты

Мир состоит из фиксированного пула чанков. Пройденные чанки получают новый абсолютный индекс, перестраивают только небольшой буфер поверхности и повторно используют заранее выделенные слоты монет, деревьев, камней, трамплинов и ускорителей. Содержимое зависит от `seed + chunkIndex`, поэтому оно воспроизводимо. Каждые 640 метров координаты сдвигаются к нулю, чтобы сохранить точность чисел.

## Перед публикацией

Текущая версия — рабочий вертикальный срез. Перед финальной модерацией стоит добавить собственные звуковые ассеты, больше вариантов чанков, иконку, обложки, локализацию описания и проверить рекламу в отладочной панели Яндекс Игр.

## Проверки

Результаты выполненных smoke-тестов находятся в `TEST_REPORT.md`. Скрипты браузерных проверок лежат в `tests/` и используют `playwright-core`; путь к Chromium при необходимости меняется внутри тестового файла.
